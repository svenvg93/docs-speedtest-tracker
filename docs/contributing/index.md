# Contributing to Speedtest Tracker

Thank you for your interest in contributing to Speedtest Tracker! We welcome contributions from the community.

## Ways to Contribute

### Report Bugs

Found a bug? Help us fix it:

1. Check [existing issues](https://github.com/alexjustesen/speedtest-tracker/issues) to avoid duplicates
2. Create a [new issue](https://github.com/alexjustesen/speedtest-tracker/issues/new) with:
   - Clear description of the bug
   - Steps to reproduce
   - Expected vs actual behavior
   - Environment details (Docker version, database type, etc.)
   - Relevant logs (remove sensitive data)

### Request Features

Have an idea for a new feature?

1. Search [existing issues](https://github.com/alexjustesen/speedtest-tracker/issues) first
2. Open a new issue describing:
   - What problem it solves
   - How it would work
   - Why it's valuable to users

### Improve Documentation

Documentation improvements are always welcome:

- Fix typos or unclear instructions
- Add missing information
- Update outdated content
- Improve examples

!!! info "Documentation Repository"
    Documentation changes should be submitted to the [speedtest-tracker-docs](https://github.com/alexjustesen/speedtest-tracker-docs) repository, not the main application repository.

### Contribute Code

Ready to write code? See our guides:

- [Development Environment](development-environment.md) - Set up your local environment
- [Pull Requests](pull-requests.md) - Submit your changes

## Before You Start

- **Check existing work**: Review [open issues](https://github.com/alexjustesen/speedtest-tracker/issues), [open pull requests](https://github.com/alexjustesen/speedtest-tracker/pulls), and [closed pull requests](https://github.com/alexjustesen/speedtest-tracker/pulls?q=is%3Apr+is%3Aclosed) to avoid duplicate work
- **Discuss major changes**: Open an issue first for significant features or architectural changes
- **Follow code style**: All code must pass linting checks

## Code of Conduct

Be respectful and constructive in all interactions. We're all here to improve the project together.

## Questions?

- Check [GitHub Discussions](https://github.com/alexjustesen/speedtest-tracker/discussions)
- Ask in an [existing issue](https://github.com/alexjustesen/speedtest-tracker/issues)
- Review the [documentation](../index.md)
